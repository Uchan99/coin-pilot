## 1. 개요
- 구현 범위 요약:
  - 마스터 체크리스트 `Priority` 열을 현재 남은 작업 순서 기준으로 재정렬하고, README backlog를 남은 작업만 남기도록 정리했다.
- 목표(요약):
  - 다음 작업 선택 기준 문서의 순서를 실제 실행 순서와 일치시킨다.
- 이번 구현이 해결한 문제(한 줄):
  - 완료 항목이 backlog 상단에 남아 있던 문서 드리프트를 해소했다.
- 해결한 문제의 구체 정의(필수: 증상/영향/재현 조건):
  - 증상: README backlog 상단에 `done` 항목 4개가 남아 있었고, 체크리스트 `Priority` 열도 현재 남은 작업 순서를 직접 반영하지 못했다.
  - 영향: 다음 실행 우선순위를 문서만 보고 판단할 때 혼선이 발생할 수 있었다.
  - 재현 조건: `README.md`와 `docs/checklists/remaining_work_master_checklist.md`의 우선순위 섹션만 보고 다음 작업을 고를 때.
- 기존 방식/상태(Before) 기준선 요약(필수):
