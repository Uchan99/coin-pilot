## 0. 트리거(Why started)
- 운영/모니터링에서 무엇이 관측됐는지:
  - 최근 작성 문서(21/24/27 계열)는 정량 근거가 보강됐지만, 과거 트러블슈팅 문서에는 정량 섹션이 누락된 케이스가 다수 존재.
- 왜 즉시 대응이 필요했는지:
  - 문서 품질 기준(99-01 정책) 일관성을 유지하려면 과거 문서도 동일 기준으로 정리할 필요가 있음.

## 1. 문제 요약
- 증상:
  - `docs/troubleshooting/*.md` 문서 간 형식 편차가 커서, 일부 문서는 해결 문제/정량 근거를 빠르게 파악하기 어려움.
- 영향 범위(기능/리스크/데이터/비용):
  - 기능: 없음(문서 품질 이슈)
  - 리스크: 트러블슈팅 재사용성 저하, 인수인계 품질 저하
- 재현 조건:
  - 정량 키워드(`정량`, `Before/After`) 기준으로 스캔하면 누락 문서 다수 확인.

## 2. 원인 분석
- 가설:
  - 정책 도입 이전에 작성된 레거시 문서는 표준 템플릿 적용이 되지 않았음.
- 조사 과정:
  - `docs/troubleshooting/*.md` 전수 스캔으로 누락 목록 식별.
- Root cause:
  - 과거 문서 작성 시점과 현재 문서 정책 사이의 표준 불일치.

## 3. 대응 전략
- 단기 핫픽스:
  - 누락 문서 말미에 `정량 증빙 상태` 섹션을 일괄 백필.
- 근본 해결:
  - 신규 문서는 템플릿 기반으로 작성, 과거 문서는 점진적으로 result/log 대조해 수치 표준화.
- 안전장치(가드레일/차단/쿨다운/timeout 등):
  - 수치 근거가 없는 항목은 임의 작성하지 않고, “측정 불가/추후 수집”으로 명시.

## 4. 구현/수정 내용
- 변경 파일:
  - `docs/troubleshooting/*.md` 중 정량 섹션 누락 문서 전수
  - `docs/checklists/remaining_work_master_checklist.md`
  - `docs/work-result/99-02_troubleshooting_quantitative_backfill_all_docs_result.md`
- DB 변경(있다면):
  - 없음
- 주의점:
  - 레거시 문서의 기존 서술은 유지하고, 정량 백필 섹션만 추가.

## 5. 검증 기준
- 재현 케이스에서 해결 확인:
  - 모든 트러블슈팅 문서에 `정량` 키워드 포함 여부 확인.
- 회귀 테스트:
  - 없음(문서 작업)
- 운영 체크:
  - 체크리스트 업데이트 로그에 본 작업 반영.
