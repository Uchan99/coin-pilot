## 1. 배경

Trade History에 실현손익 컬럼이 추가되면서 정보량은 늘었지만,
컬럼 폭이 좁아져 가독성이 떨어지고 영어 UI 용어가 혼재되어 있다.

사용자 요청:
1. Status `FILLED` 의미 명확화
2. Trade History를 Overview처럼 한국어 중심 UI로 개선
3. 가로 폭 압박 완화

---

## 2. 목표

1. Trade History 필터/컬럼/차트 라벨 한국어화
2. 기본/상세 보기 분리로 테이블 폭 최적화
3. `FILLED` 상태 의미를 페이지 내 안내로 명시

---

## 3. 아키텍처 선택 및 대안

### 선택안 (채택)
- 단일 페이지 유지 + 표시 컬럼 모드(기본/상세) 분리
- 상태값은 원본 유지, UI 설명으로 의미 전달

### 대안 1
- 테이블을 BUY/SELL 탭으로 분리
- 장점: 폭 여유
- 단점: 탐색 동선 증가

### 대안 2
- 컬럼 제거(최소 컬럼만 고정)
- 장점: 단순
- 단점: 정보 손실

### 대안 3
- 별도 상세 페이지 추가
- 장점: 확장성
- 단점: 구현/유지 비용 증가

### 트레이드오프
- 단일 페이지 모드 분리는 구현이 가볍고 학습 비용이 낮지만,
  고급 분석은 여전히 별도 페이지(Exit Analysis)와 병행하는 것이 적절하다.
