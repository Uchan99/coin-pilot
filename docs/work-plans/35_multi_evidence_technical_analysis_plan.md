## 2. 제안 전략: 다중 근거 스코어링

### 2.1 5가지 기술적 분석 피처

| # | 피처 | 정의 | 자동화 | 백테스트 |
|---|------|------|--------|---------|
| 1 | **오더블럭 (Order Block)** | 이전 캔들 몸통을 완전히 감싸는 장악형 캔들 → 지지/저항 구간 | ✅ OHLC 연산 | ✅ |
| 2 | **FVG (Fair Value Gap)** | 3캔들 사이 겹치지 않는 빈 공간(갭) → 되돌림 시 지지/저항 | ✅ High/Low 비교 | ✅ |
| 3 | **스윙 프렉탈 (Swing Fractal)** | 3캔들 중 가운데가 최고/최저 → 유동성 밀집 위치 파악 | ✅ 3캔들 비교 | ✅ |
| 4 | **추세선/채널** | 저점-저점 또는 고점-고점 연결선 + 평행선 → 지지/저항 구간 | ❌ 자동 작도 난이도 높음 | ❌ |
| 5 | **Fakeout/Trap** | 주요 지지/저항선 돌파 후 즉시 복귀 → 역진입 타점 | ⚠️ BB/스윙 저점 기반 부분 자동화 | ⚠️ |

### 2.2 적용 방식 (A+B 조합)

**Rule Engine 레이어 (자동 계산):**
- 오더블럭, FVG, 스윙 프렉탈 → 수치 피처로 계산하여 `indicators`에 추가
- Unmitigated(미해소) 구간만 유효 — 가격이 이미 침범한 구간은 무효화
- 구조적 R:R 사전 필터링 — 손익비 3:1 미만이면 스코어 탈락
- 기존 RSI/BB/MA 조건과 함께 스코어링

**AI Analyst 레이어 (LLM 판단):**
- 추세선/채널 구조, Fakeout 패턴 → OHLC 데이터 기반 AI 판단
- 프롬프트에 "다중 근거 N개 이상 충족 시 CONFIRM" 기준 반영

### 2.3 진입 규칙 (안)
```
전체 근거 점수 = Rule Engine 자동 피처 점수 + AI 판단 점수
진입 조건: 전체 근거 ≥ 2개 충족 (최소 2개 — 원칙: "근거가 2개 이상일 때만 진입")
추가 가드: 구조적 손절매 기반 R:R ≥ 3:1 필수 (원칙: "손익비 최소 3:1 이상")
```

## 3. 외부 피드백 반영 (v2 변경사항)

### 3.1 피드백 요약

| # | 지적 사항 | 핵심 | 반영 |
|---|----------|------|------|
| 1 | lookback=10은 너무 짧음 | SMC/ICT에서 신뢰도 높은 OB/FVG는 수십 캔들 전에 형성되어 미해소 상태로 유지 | lookback 100~200으로 확장 + Unmitigated 추적 |
| 2 | 구조적 손절매(Structural SL) 부재 | OB/FVG 존재만 확인하고 R:R 계산 안 하면 불량 타점 통과 | R:R 사전 필터 추가 (OB 하단 + ATR 버퍼 = SL, 스윙 고점 = TP, **기본 R:R ≥ 3.0**) |
| 3 | 시간대/세션 맥락 누락 | 아시아 세션 OB vs 뉴욕 세션 OB 신뢰도 차이 | 상위 타임프레임(HTF) 추세 정렬 필터 추가 |

### 3.2 반영 내용

**3.2.1 Unmitigated(미해소) 구간 추적**
- 초안: `lookback=10`으로 최근 OB/FVG만 감지
- 변경: `lookback=168` (1시간봉 × 168 = 7일)로 확장, 형성된 모든 OB/FVG를 배열로 관리
- 이후 가격이 해당 구간을 침범(Mitigate)했는지 추적 → 미해소 구간만 유효 반환
- 왜 7일: 현재 전략이 1시간봉 기반, 코인 시장 특성상 7일 이상 된 구간은 유효성 급감

**3.2.2 구조적 R:R 사전 필터링**
- AI 호출 전에 Rule Engine 단에서 "이 타점에 진입하면 R:R이 최소 2:1인가" 자동 계산
- 손절가(SL): OB 하단 또는 스윙 저점 - ATR(14) × 0.5 버퍼
- 익절 목표(TP): 가장 가까운 반대편 유동성 구간 (스윙 고점 또는 약세 OB)
- R:R < 3.0이면 스코어링에서 탈락 → AI에게 넘기지 않음

**3.2.3 상위 타임프레임(HTF) 추세 정렬**
- 현재 데이터: 1분봉 수집 → 1시간봉 리샘플링 (`resample_to_hourly()`)
- 추가: 4시간봉 리샘플링 함수 구현 (1분봉 원본으로부터 가능)
- `htf_trend_alignment` 피처: 4시간봉 추세 방향과 1시간봉 진입 방향 일치 여부
- 불일치 시 스코어 감점 또는 진입 차단

**3.2.4 세션 가중치 (코인 시장 적응)**
- 전통 시장: 아시아/런던/뉴욕 세션 구분
- 코인 시장: 24시간 운영이지만, 미국 시장 시간대(22:00~06:00 KST)에 변동성 집중
- 구현: OB/FVG 형성 시간대 기록 → 고변동성 시간대 형성 구간에 가중치 부여
- 백테스트에서 시간대별 OB 신뢰도를 검증하여 가중치 결정

### 15.4 핵심 발견

1. **FVG 필터가 최우수 — 단, 통계적 유의성 미확보 (⚠️ §16.1 참조)**: 승률 50.5% → 65.2% (+14.7pp), STOP_LOSS 27건 → 4건 (85% 감소), BEAR 거래 40건 → 5건으로 축소. 그러나 **N=23건(심볼당 월 1.5회)은 전략 우위(Edge) 증명에 불충분**. 90일 특정 시장 환경의 운(Variance)일 가능성 배제 불가 → 6개월~1년 확장 백테스트 필수.

2. **R:R 필터 현 구조에서 비효과적**: R:R≥3.0은 5건 모두 패배(0% 승률). 원인: 구조적 SL/TP로 진입을 필터링했지만, 실제 청산은 고정 비율 SL이 먼저 발동. 진입 필터와 청산 로직의 불일치. → **Phase 1-2에서 구조적 청산 + 포지션 사이징 역산 백테스트 필요**

3. **HTF 정렬은 역효과**: BEAR 거래 40→8건으로 줄였지만 승률 34.9%로 하락. BEAR 레짐에서도 유효한 진입이 존재했는데 이를 제거.

4. **OB/프렉탈 필터링 효과 미미**: OB 34% 제거했지만 PnL 개선 미미, 프렉탈 13% 제거로 사실상 무필터.

5. **2ev 시나리오 = R:R 시나리오와 동일**: R:R가 지배적 제약 조건, 다른 근거가 추가적 필터링 효과를 내지 못함.

### 15.5 다음 단계

- **백테스트 기간 확장 (최소 6개월~1년)**: FVG 필터 N=100건 이상 확보 후 단독 우위 재검증 (§16.1)

## 16. 4차 외부 피드백 반영 (v3.1 변경사항)

### 16.1 통계적 유의성 결여 (N=23의 함정)

**지적**: FVG 필터 N=23건(심볼당 월 1.5회)으로는 65.2% 승률이 전략 우위(Edge)인지 시장 운(Variance)인지 증명 불가. 23건으로 핵심 필터를 확정하는 것은 과최적화(Curve-fitting).

**수용**:
- FVG를 "확정된 핵심 필터"가 아닌 **"유망 후보, 추가 검증 필요"** 로 격하
- 백테스트 기간을 **최소 6개월~1년으로 확장**하여 N≥100건 확보 후 재검증
- 현재 DB에 충분한 데이터가 있는지 확인 필요 (market_data 테이블 최대 기간)
- 검증 기준: N≥100에서도 승률 60% 이상 + PnL 개선 유지 시 채택

### 16.2 포지션 사이징 — 현물(Spot) 전용 비중 조절 로직

**4차 지적**: `Position Size = (Equity × 0.02) / Risk` 공식 + 상한/하한 가드 제안.

**5차 지적 — 수학적 모순 (§17.1)**: 이 공식은 현물 시장에서 성립 불가.
- Risk=0.5% → Position = Equity × 4배 (400%) → 현물에서 레버리지 불가, 주문 실패
- max_per_order=30% 캡 적용 시 실제 리스크 = 0.30 × 0.005 = 0.15% → "고정 2% 리스크" 허구
- **결론: 레버리지 없는 현물 시장에서 고정 리스크 모델은 타이트한 SL에서 작동 불가**

**수정 — 현물 전용 캡 역산 모델 (v4.0 최종, §19.2)**:

```
현물 포지션 사이징 로직 (v4.0 최종):
1. 역산 공식 (타점별 리스크 정규화):
   - calculated_size = (equity × 0.02) / structural_sl_pct
   - 목적: SL 넓은 타점은 비중 축소, SL 좁은 타점은 비중 확대

2. 현물 캡 (레버리지 불가 제약):
   - max_order = equity × regime_ratio × symbol_multiplier (기존 Rule Engine)
   - final_size = min(calculated_size, max_order)
   - SL 타이트(0.5%) → 계산값 400% → 캡 → max_order (2% 리스크 미달, 수용)
   - SL 넓음(4%) → 계산값 50% → 캡 → max_order
   - SL 매우 넓음(8%) → 계산값 25% → 그대로 사용 (자동 비중 축소)

3. 극단값 가드:
   - structural_sl_pct > 5.0%: 진입 스킵 (비정상적으로 넓은 SL)
   - structural_sl_pct < 0.3%: 진입 스킵 (스탑 헌팅 고위험)

4. 안전장치:
   - 상한: max_order 초과 불가
   - 하한: 업비트 최소 주문 금액 5,000원 미달 시 진입 스킵
```

**핵심**: 타점마다 SL 폭이 달라도 감수 리스크가 정규화됨. 현물 캡으로 레버리지 불가 제약 해소.

**변경 이력**:
- v3: 고정 2% 역산 → v3.2: 계단식 (3%/5%/0.7) → v3.3: ATR 연속 스케일링 → **v4.0: 캡 역산 `min((Eq×0.02)/SL%, max_order)` (최종)**

### 16.3 청산 철학 자기모순 해소 — 구조적 트레일링으로 수정

**지적**: "구조적 SL/TP 저장 + Parabolic SAR/ATR 동적 트레일링"은 근본적으로 충돌.
- SMC/ICT 이론: 올바른 익절 = 반대측 유동성(스윙 고점/약세 OB)
- PSAR/ATR: 추세 추종형 지표 → 노이즈에 반응하여 구조적 목표 도달 전 중간에 절단

**수용 — 트레일링 스탑 기획 전면 수정**:

| 기존 계획 (폐기) | 수정 계획 |
|-----------------|----------|
| Parabolic SAR 기반 트레일링 | **구조적 트레일링 (Structural Trailing)** |
| ATR 기반 동적 트레일링 | 가격이 목표로 이동 중 **새 스윙 저점 형성 시 SL을 해당 위치로 이동** |

구현 로직 (Phase 1-2 백테스트):
```
1. 진입 시: SL = 구조적 SL (OB 하단 or 스윙 저점 - ATR 버퍼)
2. 보유 중: 유의미한 스윙 저점(아래 정의) 형성 시 → SL을 해당 위치로 상향
3. 익절: 구조적 TP (스윙 고점 or 약세 OB) 도달 시 청산
4. 폴백: 구조적 TP 미도달 + 시간 초과 시 → 기존 TIME_LIMIT 청산 유지
```

**⚠️ 5차 피드백 반영 — 스윙 저점 정의 엄격화 (§17.2)**:

1시간봉 3캔들 프렉탈을 그대로 쓰면 마이크로 노이즈 저점이 수시로 발생하여 SL이 너무 빨리 올라감 → 스탑 헌팅으로 조기 청산.

트레일링용 유의미한 스윙 저점 조건 (아래 중 하나 이상 충족):
```
A) 4시간봉 기준 스윙 저점: 1시간봉이 아닌 4시간봉에서 3캔들 프렉탈 형성
B) BOS (Break of Structure) 동반: 이전 고점을 명확히 갱신한 파동의 저점
   - 즉, 새 고점(Higher High) 확인 후 그 직전 되돌림 저점만 유효
C) 거래량 확인: 해당 저점 캔들의 거래량 ≥ 20봉 평균의 1.2배
```

백테스트에서 A/B/C 각 조건 및 조합별 트레일링 성과를 비교하여 최적 정의 결정.

**~~6차 피드백: 분할 익절 + 본절 이동~~ → 7차 피드백으로 분할 익절 폐기, 8차 피드백으로 BE 부활 (§19.1, §20.3)**:

분할 익절(30% at 50% TP)은 R:R 1:3 기준 1승 +3R → +0.45R로 기댓값을 파괴 → **분할 익절 폐기 유지**.
단, 본절 이동(BE)은 분할 익절과 성격이 다름 — 물량 변경 없이 SL만 이동하므로 EV 파괴 없이 꼬리 리스크 방어 가능 → **BE 부활** (§20.3).

```
구조적 트레일링 + BE 방어 로직 (v4.1 최종, Phase 1-2 백테스트):
1. 진입 시: 전체 물량 매수, SL = 구조적 SL, TP = 구조적 TP
2. BE 트리거: 가격이 1.5R 수익 구간 도달 시 → SL을 진입가(Entry)로 이동 (물량 100% 유지)
3. 보유 중: 유의미한 스윙 저점(위 A/B/C 조건) 형성 시 → SL을 해당 위치로 상향
   - BE 이후에도 구조적 트레일링 계속 작동 (SL은 항상 상향만)
4. 익절: 구조적 TP 도달 시 전량 청산
5. 폴백: 구조적 TP 미도달 + 시간 초과 → TIME_LIMIT 청산 유지
- 분할 익절 없음 (R:R 기댓값 보존)
- BE 트리거 임계값(1.0R/1.5R/2.0R)은 백테스트에서 비교 결정
```

**백테스트 비교 항목 유지**: 분할(30% at 50% TP) vs 비분할(구조적 트레일링+BE)을 백테스트에서 데이터로 검증. 분할이 EV 측면에서 우위인 경우(TP 도달률이 극히 낮은 시장)만 예외적 채택.

## 17. 5차 외부 피드백 반영 (v3.2 변경사항)

### 17.1 현물(Spot) 시장과 고정 리스크 모델의 수학적 모순

**지적**: `Position Size = (Equity × 0.02) / Risk` 공식은 레버리지 불가 현물 시장에서 성립 불가.
- Risk=0.5% → Position = Equity × 400% → 현물에서 주문 불가
- max_per_order=30% 캡 적용 시 실제 리스크 = 0.15% → "고정 2%" 허구
- **고정 리스크 모델은 타이트한 SL + 현물 조합에서 수학적으로 파탄**

**수용**: §16.2를 전면 수정.
- 고정 리스크(2%) 역산 모델 **폐기**
- **현물 전용 조건부 비중 축소 모델** 채택:
  - 기본 비중 유지 (기존 Rule Engine 사이징)
  - SL 거리가 넓을 때(>3%)만 비중 축소, 극단적 SL(<0.3% 또는 >5%)은 진입 스킵

### 17.2 구조적 트레일링 — 스윙 저점 정의의 빈도 과소평가

**지적**: 1시간봉 3캔들 프렉탈을 스윙 저점으로 쓰면 마이크로 노이즈 저점이 수시로 발생.
- 상승장에서도 매시간 얕은 저점 형성 → SL이 너무 빨리 올라감
- 다음 캔들 꼬리에 스탑 헌팅 → 턱없이 짧은 수익으로 조기 청산

**수용**: §16.3에 스윙 저점 엄격 정의 추가.
- 트레일링용 유의미한 스윙 저점 3가지 후보 조건:
  - A) 4시간봉 기준 프렉탈 (1시간봉 노이즈 필터링)
  - B) BOS(Break of Structure) 동반 (새 고점 갱신 후 되돌림 저점만 유효)
  - C) 거래량 확인 (해당 저점 캔들 거래량 ≥ 20봉 평균 × 1.2)
- Phase 1-2 백테스트에서 A/B/C 각 조건 및 조합별 성과 비교

### 17.3 시간 Pruning과 SMC 이론의 충돌

**지적**: "168시간 지나면 자동 만료"는 ICT/SMC 이론과 정면 충돌.
- 가장 강력한 반전 포인트 = 수일~수주 전부터 미해소된 일봉/주봉 단위 OB/FVG
- 시간 경과만으로 만료시키면 차트상 가장 강력한 자석(Magnet)을 시스템이 스스로 삭제

**수용**: §16.4에서 시간 Pruning 폐기.
- **탐색 기간(Detection) ≠ 수명(Lifespan)** 원칙 확립
  - `lookback=168`은 신규 OB/FVG를 탐색하는 윈도우 범위
  - 한 번 발견된 Unmitigated 구간의 수명은 **무기한**
- 만료 조건은 오직 2가지:
  1. **가격 침범(Mitigation)**: 종가가 OB/FVG 영역을 침범

## 18. 6차 외부 피드백 반영 (v3.3 변경사항)

### 18.1 ~~매직 넘버(Magic Number) 함정 — ATR 연속 스케일링으로 대체~~ → v4.0에서 캡 역산으로 재수정 (§19.2)

**지적**: §16.2의 "SL 3~5% → ×0.7 축소"는 임의의 매직 넘버. 시장 변동성 체제가 바뀌면(저변동→고변동) 시스템이 장 전체에 걸쳐 70% 비중으로만 거래 → 기댓값 붕괴.

**수용**: 계단식(Step-function) 하드코딩 **폐기** → ATR 기반 연속 스케일링 채택.

```
vol_scale = clamp(target_atr / current_atr, 0.5, 1.5)
final_size = base_size × vol_scale
```

- `target_atr`: ATR(14)의 90일 이동 평균 = 시장의 "기준" 변동성
- `current_atr`: 현재 ATR(14) = 실시간 변동성
- 변동성 높으면 비중이 연속적으로 축소, 낮으면 확대 (이진 점프 없음)
- clamp(0.5, 1.5): 극단 방지 — 최소 절반, 최대 1.5배

**장점**: 매직 넘버 없이 시장 적응형. 변동성 체제 전환 시 자동 조절.

### 18.2 ~~무기한 수명의 상태 관리 붕괴 — 구조적 무효화~~ → v4.0에서 공간적 상한선으로 대체 (§19.3)

**지적**: 무기한 수명 + 가격 거리 가리기(Hide)만으로는 40% 하락장에서 과거 상승장의 미해소 OB/FVG가 수백 개 누적 → 메모리 누수 + 상태 복잡도 폭증.

**수용**: 가격 거리 Pruning 외에 **구조적 무효화(Structural Invalidation)** 룰 추가.

```
구조적 무효화 규칙:
- 일봉(1D) 기준 메이저 지지선 하방 BOS (종가 하향 돌파) 발생 시:
  → 해당 지지선 위에 남아있는 1시간봉 강세 OB/FVG를 전부 영구 삭제
- 일봉(1D) 기준 메이저 저항선 상방 BOS 발생 시:
  → 해당 저항선 아래의 1시간봉 약세 OB/FVG를 전부 영구 삭제
- 근거: 추세 전환(BOS)으로 해당 가격 레벨의 구조적 의미가 소멸
```

**효과**: 추세 전환 시 과거 구간 일괄 정리 → 배열 크기 자연 억제, 상태 관리 단순화.

### 18.3 ~~BOS 트레일링의 타이밍 지연 — 분할 익절 + 본절 이동~~ → v4.0에서 구조적 트레일링 단일로 복원 (§19.1)

**지적**: BOS 확인(이전 고점 종가 돌파) 시점에는 가격이 이미 고점 부근. 뒤늦게 SL 상향해도 깊은 되돌림에 미실현 수익 보호 불가.

**수용**: 트레일링 단독 의존 **폐기** → 분할 익절(Scaling Out) + 본절 이동(Breakeven) 병행.

```
복합 청산 로직 (Phase 1-2 백테스트):
1. 진입: 전체 물량 매수, SL = 구조적 SL
2. TP 50% 도달: 30% 물량 즉시 익절 → 확정 수익 확보
3. BE 이동: 잔여 70% 물량의 SL을 진입가로 상향 → 손실 위험 0
4. BOS 트레일링: BE 보호 상태에서 BOS 기반 SL 추가 상향 시작
5. 최종 청산: 구조적 TP 도달 or 트레일링 SL 히트 or TIME_LIMIT
```

**핵심**: BOS 확인 지연에 의한 손실 위험을 사전에 제거. "수익 확정 → 무위험 → 추가 수익 추구" 3단계.

### 18.4 콜드 스타트(Cold Start) — 탐색 기간과 초기화 분리

**지적**: lookback=168로 탐색 + 발견 후 무기한 유지인데, 시스템 최초 가동 시 168시간만 스캔하면 10일 전 강력 일봉 FVG를 영원히 인지 못함.

**수용**: 초기화(Initialization) 로드 범위와 매 캔들 갱신(Incremental) 범위 분리.

```
데이터 로드 전략:
1. 매 캔들 갱신 (Incremental):
   - lookback=168시간 (기존)
   - 새로 형성된 OB/FVG만 기존 배열에 추가 (append)

2. 시스템 최초 가동 (Cold Start):
   - 과거 90일 1시간봉 일괄 스캔 → 초기 Master State 구축
   - 모든 Unmitigated 구간을 식별하여 배열 초기화
   - 이후 Incremental 모드로 전환

3. 백테스트:
   - 전체 데이터 일괄 스캔 (기존 방식 유지)
   - 시뮬레이션 시작점에서 lookback 범위의 초기 상태 구축 후 순차 진행
```

**효과**: 과거 강력 구간 누락 방지 + 매 캔들 갱신 시 연산 부하 최소화.

## 19. 7차 외부 피드백 반영 (v4.0 변경사항)

**배경**: v3.3(6차 피드백)에서 도입한 ATR 연속 스케일링(§18.1), 구조적 무효화(§18.2), 분할 익절+본절 이동(§18.3)에 대한 근본적 재검토.

### 19.1 분할 익절 EV 붕괴 — 구조적 트레일링 + BE 방어로 복원 (8차 피드백 수정, §20.3)

**지적**: 30% 분할 익절(50% TP 도달 시)이 R:R 기댓값을 파괴.
- R:R 1:3 기준, 1승 +3R → 분할 시 +0.45R로 축소
- 나머지 70%가 본절(BE)에서 스탑 → 0.7R × 0 = 0 기여
- 분할 30%의 기여: 0.3R × 1.5R(50% TP) = +0.45R
- 비분할 시 1승 기여: 1.0R × 3R = +3R → **6.67배 차이**

**수용**:
- ~~분할 익절(30% at 50% TP)~~ 폐기 (§18.3 deprecated) — EV 파괴, 폐기 유지
- **본절 이동(BE)은 부활** (§20.3): 분할 익절과 성격이 다름 — 물량 변경 없이 SL만 이동하므로 EV 파괴 없이 꼬리 리스크 방어
- **구조적 트레일링 + BE 방어** 방식: 전량 보유 → 1.5R 도달 시 SL→진입가 → 스윙 저점(A/B/C) SL 상향 → 구조적 TP 전량 청산
- SMC/ICT 프레임워크 정합성 + 수학적 기댓값 보존 + 꼬리 리스크 방어
- **단, 분할 vs 비분할은 백테스트에서 데이터로 최종 검증** (TP 도달률이 극히 낮은 시장이면 예외 가능)

### 19.2 ATR 스케일링 → 캡 역산(Capped Inverse) 부활

**지적**: ATR 연속 스케일링(§18.1)은 "시장 전체" 변동성만 보정. 동일 시장에서 SL 1% 타점과 SL 4% 타점의 리스크 4배 차이를 해결하지 못함.

**핵심 문제**: 같은 ATR 환경에서도 개별 타점의 구조적 SL 거리가 다름 → per-trade 리스크 정규화가 핵심

**수용 — 캡 역산 공식**:
```
Calculated = (Equity × 0.02) / structural_sl_pct
Final = min(Calculated, max_per_order)   ← 현물 캡
```

동작 예시:
| SL 거리 | 계산값 | 캡 적용 | 실제 리스크 |
|---------|--------|---------|------------|
| 0.5% (타이트) | 400% | 30% (캡) | 0.15% (2% 미달, 수용) |
| 2.0% (보통) | 100% | 30% (캡) | 0.6% |
| 4.0% (넓음) | 50% | 30% (캡) | 1.2% |
| 8.0% (매우 넓음) | 25% | 25% (그대로) | 2.0% (목표 달성) |

- **타이트 SL**: 캡이 걸려 2% 리스크 달성 불가 → 수용 (현물 제약)
- **넓은 SL**: 비중 자동 축소 → 리스크 자연 제한
- 5차 피드백(§17.1)의 현물 모순도 이 공식으로 해결. 공식 자체 폐기(v3.2)는 과잉 대응이었음
- ~~ATR 연속 스케일링~~ 폐기 (§18.1 deprecated)
- ~~계단식 조건부 비중 축소~~ 폐기 (§17.1 deprecated)

### 19.3 구조적 무효화 → 공간적 상한선(Spatial Capacity) 5+5

**지적**: 1D BOS 기반 구조적 무효화(§18.2)는 그 자체가 별도 프로젝트급 복잡도. BOS 감지 정확도, 타임프레임 선택, 엣지 케이스 처리 등 오버엔지니어링.

**수용 — 공간적 상한선**:
```

- 위로 가장 가까운 미해소 약세 OB/FVG: 최대 5개
- 아래로 가장 가까운 미해소 강세 OB/FVG: 최대 5개
- 가격 이동 시 멀어진 구간은 큐에서 자연 밀려남
```

**장점**:
- O(1) 연산, 상태 관리 단순
- 추세 전환/급등락 시에도 자연 정리 (새 구간이 기존 구간을 밀어냄)
- 시간 만료 불필요, 구조적 무효화 불필요
- ~~구조적 무효화(1D BOS)~~ 폐기 (§18.2 deprecated)

## 20. 8차 외부 피드백 반영 (v4.1 변경사항)

**배경**: v4.0 기획안의 코드 레벨 구현 시 드러나는 3가지 논리적 맹점에 대한 검토.

### 20.1 공간적 상한선 5+5의 군집화(Clustering) 오류

**지적**: 원웨이 급락 시 상위 1-2% 구간에 마이크로 FVG가 밀집 형성 → 5% 아래의 메이저 OB가 거리 순위 6위로 밀려나 영구 삭제.

**수용 — Zone 병합 전처리** (코드 레벨 구현):
```
전처리: 서로 ATR(14)×0.3 이내에 인접한 OB/FVG → 하나의 Zone으로 병합
병합 후 5+5 상한선 적용
→ 급락 시 마이크로 FVG 5개가 1개 Zone으로 축소, 메이저 지지선 보존
```

- 고정 0.5% 대신 ATR 기반 동적 임계값 → 매직 넘버 회피
- 백테스트에서 병합 유무 비교로 최종 효과 검증
- 기획 수정 불필요, 코드 구현 시 반영

### 20.2 R:R 사전 필터의 슬리피지 페널티 누락

**지적**: SL 0.4% 타점에서 시장가 손절 시 수수료(0.05%) + 슬리피지(0.1~0.2%) → 실제 손실 0.6~0.65%. 사전 R:R 3.0이 실전 R:R ~1.8로 반토막.

**수용 — Net R:R 계산** (코드 레벨 구현):
```python
estimated_slippage = 0.15  # 수수료 + 호가창 슬리피지 (%)
net_risk = structural_sl_pct + estimated_slippage
net_rr = structural_tp_distance / net_risk
# net_rr >= 3.0 일 때만 진입
```

- 기존 백테스트에 수수료는 반영되어 있으나, R:R 사전 필터에는 미반영 — 이 간극을 메움
- 기획 수정 불필요, 코드 구현 시 반영

### 20.3 본절(BE) 이동 부활 — 분할 익절과 분리

**지적**: 7차 피드백에서 분할 익절과 본절 이동을 일괄 폐기한 것은 과잉 대응. 구조적 트레일링만으로는 원웨이 상승 후 V자 급락(스윙 저점 미형성) 시 수익 전체 반납 → -1R 손실.

**핵심 구분**:
| 메커니즘 | 물량 변경 | EV 영향 | 목적 |
|---------|----------|---------|------|
| 분할 익절 (30% at 50% TP) | ✅ 물량 감소 | ❌ EV 파괴 (+3R→+0.45R) | 수익 확정 |
| 본절 이동 (BE) | ❌ 물량 유지 | ⚠️ 미미한 EV 비용 | 꼬리 리스크 방어 |

**BE의 EV 시나리오 분석**:
| 시나리오 | BE 있음 | BE 없음 | 차이 |
|---------|---------|---------|------|
| 1.5R 도달 → TP(3R) 달성 | +3R | +3R | 동일 |
| 1.5R 도달 → 진입가 복귀 → 재상승 TP | 0R (조기 청산) | +3R | BE 비용 |
| 1.5R 도달 → V자 급락 → 원래 SL | 0R (BE 방어) | -1R | **BE 이익 (+1R)** |
| 1.5R 미도달 → SL 청산 | -1R | -1R | 동일 |

**수용 — BE 트리거 부활** (기획 수정, §16.3 반영):
```
BE 방어 로직:
- 가격이 1.5R 수익 구간 도달 시 → SL을 진입가(Entry)로 이동
- 물량 100% 유지 (분할 없음)
- 이후 구조적 트레일링 계속 작동 (SL은 항상 상향만)
- BE 트리거 임계값(1.0R/1.5R/2.0R)은 백테스트에서 비교 결정
```

- ~~분할 익절~~ 폐기 유지 (EV 파괴)
